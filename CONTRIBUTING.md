# Contributing to Hydro-Resource-Hub

Thank you for your interest in contributing! This project thrives on community contributions from engineers, researchers, and students in the water resources field. Please take a moment to read these guidelines before submitting a pull request.

---

## How to Add a Resource

1. **Fork** this repository and create a new branch from `main`:
   ```bash
   git checkout -b add/your-resource-name
   ```

2. **Edit `README.md`** and add your resource to the appropriate category table. Use the following format:
   ```markdown
   | [Resource Name](https://resource-url.example) | A concise, one-sentence description of the resource. |
   ```

3. **Open a Pull Request** against the `main` branch with a clear title such as `Add [Resource Name] to [Category]`.

---

## Contribution Rules

### Quality Standards
- **Freely accessible**: Prefer free, open-source, or publicly accessible resources. Commercial tools may be included if they offer a meaningful free tier or academic license.
- **Relevant**: Resources must be directly relevant to water resources engineering, hydrology, hydraulics, or closely related geospatial/environmental fields.
- **Working links**: Ensure all URLs are active and point to the official source or documentation.
- **One resource per PR**: Keep pull requests focused — one new resource per PR makes reviewing faster and easier.

### Formatting Guidelines
- Match the existing table format in `README.md` exactly.
- Resource names should use **Title Case**.
- Descriptions should be a single, concise sentence (under 120 characters) ending with a period.
- If a new category is needed, discuss it in an Issue first before adding it.

### What Will Not Be Accepted
- Duplicate entries (check existing entries before submitting).
- Broken or unreachable links.
- Resources that are purely commercial with no free access.
- Self-promotional links unrelated to the field.
- Changes to existing descriptions without a clear reason stated in the PR.

---

## Reporting Issues

If you find a broken link, outdated information, or want to suggest a new category, please [open an Issue](../../issues/new) describing the problem or suggestion.

---

## Code of Conduct

Be respectful and constructive. This is a collaborative resource for the engineering and research community. Contributions of all skill levels are welcome.

---

Thank you for helping make Hydro-Resource-Hub more useful for everyone! 💧
